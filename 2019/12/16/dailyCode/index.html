关于前端起服务：

前端为什么要起服务？
为了解决跨域；直接通过浏览器打开是类似file:///的访问形式，即file协议，起服务运行在服务器环境才可能解决跨域问题。本地页面ajax()请求本地页面，须通过服务器环境运行
有什么方便的起服务的方法？
用http-server可以方便地起本地服务。
命令行：http-server -c -l

terminal更改文件夹性质的命令：

sudo chown -R userName:staff a「改变a文件夹的性质，从sudo改成userName用户就能操作」
ctrl+R+字符：搜索使用过的字符的语句，搜索完按右箭头即可定位到该语句
history：查找所有历史语句
grep:分组查找
ls -l：查看文件夹下所有文件，并展示读写权限
dig 域名：没接入cdn的域名可以查看域名对应的vip

cur测试页面（一定要带-I，不然返回的是html）：

curl -H ‘Host:页面host’ ‘http://具体vip/finance-gold/msjgold/homepage?orderSource=4’ -I
curl -H ‘Host:m.jdjygold.com’ ‘http://61.49.99.30/finance-gold/msjgold/homepage\?orderSource\=4' -I
curl -v url可以用来测一个地址是否能正常访问(如果https证书不支持，会返回subjectAltName doses not match)

ios低版本手机滑动fixed按钮跟着滚动的问题：

设置了-webkit-overflow-scrolling: touch;解决方法：body、html、wrap、content均设置width：100%，height：100%，在content上加webkit-overflow-scrolling: touch;既可以顺滑滚动，又可以fixed不跟随改动
——》引入其他问题：浏览器弹性滚动问题
把 -webkit-overflow-scrolling: touch;放在小div而不是放在body上的问题：快速上拉页面[还在弹性滑动的过程中]迅速得再向上拉，会出现页面整个被拉动的情况[上面是空白的样式]；拉动的问题和fixed+touch的问题无法同时解决，只能通过引入isScroll之类的问题解决
———》关联的问题：safari渲染问题
单页面应用画echart图的问题：从有echart的的页面跳入下一个页面[滑动一段距离之后]，再back的时候echart图出不来了。原因：body页面很长可以滑动，滑完点击跳入下一个页面，再返回的时候，页面渲染机制在把页面上滑之后就停止了，不会在echart图数据返回的时候渲染页面，因此白屏。如果把滑动限制在div里(body设置width:100%,height:100%)里可以解决这个问题。但是滑动限制div里会引起上面那个问题。
